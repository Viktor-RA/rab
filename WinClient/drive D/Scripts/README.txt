Задача мониторить состояние двух контроллеров доступа PARSEC. С различными API и т.п. я не совладал и пошел другим путём. Софт пишет логи, в sql таблицу. Оттуда мы и выдергиваем необходимые нам сведения. Можно конечно вычитывать все события: проходы, бэкапы, выполненные скрипты, но это морока и нет такой необходимости в моём случае. 
В логах мои железки отображаются с ID  
E7E3AEDA-6093-44A8-8A1B-0B0B0033A84C
9B884D40-BD58-460B-9B09-27604C200746
эти ID необходимо заменить на свои в фалах SELECT_1 и SELECT_2 соответсовенно. Хотя у Вас не обязательно будет два контроллера и придётся много что где изменять))
Узнать свои ID можно поизучав вывод после запуска SELECT_65667.bat в файле 65667.log

!!! В BAT-файлах "PARSECPC" заменить на имя своей рабочей станции

На машине, на которой крутится серверная часть ParsecNet OFFICE (в частности SQL сервер, устанавливаемый вместе с софтом) лежит содержимое всей этой папки. Запущен от имени администратора и постоянно выполняется RUN.bat
По сути он периодически вызывает скрипт запроса последней стоки логов с выборкой по конкретному контроллеру, при обнаружении изменений дописывает полную строку лога в файл hardware.log, производит запись номера строки и кода состояния в файлы hw(номер)(тип запроса).log, откуда агентом выдёргиваем парсером состояние контроллера.

IШаблонов нет, я создавал по три элемента на каждый контроллер:

наличие связи
тип Zabbix-агент
ключ vfs.file.contents[D:\Scripts\hw1CONNECT.log]
Тип - целое положительное
Предобработка: регулярное выражение: (?<= )\d+   /0

наличие питания
тип Zabbix-агент
ключ vfs.file.contents[D:\Scripts\hw1POWER.log]
Тип - целое положительное
Предобработка: регулярное выражение: (?<= )\d+   /0

Крышка закрыта
тип Zabbix-агент
ключ vfs.file.contents[D:\Scripts\hw1COVER.log]
Тип - целое положительное
Предобработка: регулярное выражение: (?<= )\d+   /0

для второго контроллера меняется толко ключ:
vfs.file.contents[D:\Scripts\hw2(БЛАБЛАБЛА).log]

Мне известны такие состояния контроллеров, которые меня и интересуют:
65540 - Связь потеряна
65667 - Связь восстановлена
65324 - Питание отключено
65325 - Питание восстановлено
66308 - Крышка открыта
66309 - Крышка закрыта

так-же отдельным элементом мониторится процесс cmd.exe на запущенность с соответствующим триггером. В случае ошибки выполнения bat файла процесс закроется.
